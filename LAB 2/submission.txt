Objective

To understand the working of LEX (Lexical Analyzer Generator) by writing regular expressions for tokens, generating scanners, experimenting with different inputs, and modifying lex scripts to control token recognition, lexeme reporting, and counting.
---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------


In this laboratory assignment, we studied the use of LEX, a lexical analyzer generator, to understand how tokens are recognized using regular expressions. The objective was to gain hands-on experience in writing lex scripts, generating scanners, and analyzing their behavior on different input strings. Initially, tokens were defined over the alphabet {a, b, c}, where Token 1 corresponds to the regular expression b*c+ and Token 2 corresponds to a*b. Using these definitions, we analyzed how a DFA recognizes valid substrings from the input using the longest match rule. In P2, a scanner was generated using the given regex1.l script. The lex script was processed using the lex command to generate lex.yy.c, which was then compiled using GCC. When the scanner was executed on different input files, it was observed that only the first valid token was recognized.
